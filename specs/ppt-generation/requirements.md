# AI 自动生成 PPT - 需求文档

## 1. 项目概述

### 1.1 背景
在现有 AI 创意工坊的基础上，新增 AI 自动生成 PPT 功能，复用项目已有的多 LLM 后端、AI 图片生成、Pipeline 架构等核心能力，实现从文档到精美 PPT 的全自动转换。

### 1.2 核心价值主张
**"精美、有人味、充分利用优势"** — 这是本功能最重要的设计原则：

1. **精美**：排版专业、配色讲究、视觉层次清晰
2. **有人味**：不模板化、不千篇一律、有设计感和温度
3. **充分利用优势**：
   - AI 定制配图（SiliconFlow/阿里云万相/Together.ai/本地 Stable Diffusion）
   - 多 LLM 选择（Gemini 免费/DeepSeek 便宜/OpenAI 高质量/Ollama 本地）
   - 成熟的 Pipeline 架构（断点续传、任务队列、错误恢复）

### 1.3 输入与输出
- **输入**：文档/文本（txt、markdown、或直接粘贴文字）
- **输出**：高质量 .pptx 文件（可在 PowerPoint / Keynote / WPS 中打开）

## 2. 功能需求

### 2.1 核心流程（6 阶段 Pipeline）

#### 阶段 1：文档分析
**目标**：深度理解输入文档的内容、结构、受众。

- 提取核心主题（1-2 句话概括）
- 识别文档类型（工作汇报、产品介绍、教学课件、技术分享、创意提案等）
- 分析受众（商务、技术、教育、创意等）
- 提取关键信息点（核心观点、数据、案例、结论）
- 检测文档结构（是否有明确章节、是否有数据表格、是否有引用内容）

**输出**：
```python
{
  "theme": "产品市场分析报告",
  "doc_type": "business_report",
  "audience": "business",
  "key_points": ["市场规模增长30%", "竞争对手分析", "产品定位策略"],
  "structure": {"has_sections": True, "has_data": True, "has_quotes": False},
  "tone": "professional"  # professional / creative / educational / casual
}
```

#### 阶段 2：大纲生成
**目标**：生成 PPT 大纲，确定每页的类型、内容要点、视觉风格。

- 根据文档长度和内容密度，确定页数（8-30 页）
- 为每页分配类型（封面、章节分隔、内容页、数据展示、引用、结束页等）
- 确保布局多样性：同一 PPT 中不能连续 3 页使用相同布局
- 设计"节奏感"：文字密集页 → 图片页 → 数据页 → 短句页，保持交替
- 为每页生成标题和要点（bullet points，3-5 个）

**输出**：
```python
{
  "total_pages": 15,
  "estimated_duration": "8-10分钟演讲",
  "slides": [
    {
      "page_number": 1,
      "type": "title_hero",
      "title": "2024 年产品市场分析报告",
      "subtitle": "数据驱动的战略洞察",
      "layout": "title_hero",
      "needs_image": True,
      "image_style": "abstract_business"
    },
    {
      "page_number": 2,
      "type": "section_divider",
      "title": "市场现状",
      "layout": "section_divider",
      "needs_image": False
    },
    {
      "page_number": 3,
      "type": "content",
      "title": "市场规模持续增长",
      "key_points": ["2024年市场规模达500亿", "同比增长30%", "预计2025年突破650亿"],
      "layout": "text_left_image_right",
      "needs_image": True,
      "image_prompt": "市场增长趋势，向上箭头，商务风格"
    },
    // ...
  ]
}
```

#### 阶段 3：内容创作
**目标**：为每页生成精炼的文案（标题、要点、演讲备注）。

- 标题提炼：简洁有力，不超过 15 个汉字
- 要点提炼：每条不超过 30 字，3-5 个要点
- 避免"AI 味"：
  - 不用"赋能"、"闭环"、"抓手"等空洞词汇
  - 不用"首先、其次、最后"等刻板结构（至少一半页面要打破）
  - 数据具体化：不说"显著增长"，而是"增长 30%"
  - 语言有温度：不是"用户痛点"，而是"用户最头疼的是"
- 生成演讲备注（Speaker Notes）：为演讲者提供详细解说词

**输出**：
```python
{
  "page_number": 3,
  "title": "市场规模持续增长",
  "content": {
    "bullet_points": [
      "2024年市场规模达500亿元",
      "同比增长30%，超出预期",
      "预计2025年突破650亿"
    ],
    "speaker_notes": "根据最新行业数据，市场规模在过去一年内实现了30%的增长..."
  }
}
```

#### 阶段 4：设计编排
**目标**：为每页选择布局、配色、字体、装饰元素，确保多样性和节奏感。

**4.1 布局选择（至少 10 种）**
- `title_hero`：封面页（大标题+副标题+背景图）
- `section_divider`：章节分隔页（大标题居中+装饰元素）
- `text_left_image_right`：左文右图（60/40 分割）
- `image_left_text_right`：左图右文（40/60 分割）
- `full_image_overlay`：全屏图+文字叠加（深色半透明遮罩）
- `three_columns`：三栏并列（适合对比、列举）
- `quote_page`：引用/金句页（大字号居中+引用框）
- `data_highlight`：数据突出展示（巨大数字+小字说明）
- `timeline`：时间线/流程图（横向或纵向）
- `bullet_with_icons`：要点+图标（每个要点左侧配图标）
- `comparison`：对比页（左右两栏，A vs B）
- `closing`：结束页（感谢+联系方式）

**4.2 配色方案（非呆板商务风）**
- 不是"纯蓝白"，而是：
  - 主色：深蓝 #1A237E / 深绿 #2E7D32 / 深紫 #4A148C / 深橙 #E65100
  - 辅色：金色 #FFD700 / 青绿 #00BCD4 / 玫瑰金 #B76E79 / 浅灰 #ECEFF1
  - 强调色：高亮黄 #FDD835 / 珊瑚红 #FF6F61
- 渐变色使用：标题背景、分隔线、图形元素
- 色彩饱和度：60-80%（不是 100% 纯色，显得更高级）

**4.3 字体层级（4 级）**
- 一级标题：44pt，粗体，主色
- 二级标题：32pt，半粗体，主色
- 正文：20pt，常规，深灰 #424242
- 注释：14pt，常规，中灰 #757575

**4.4 留白与间距**
- 页边距：左右 80px，上下 60px
- 标题与正文间距：40px
- 要点之间间距：24px
- 大量留白 = 高级感（内容不超过页面 60%）

**4.5 装饰元素（微妙点缀，不堆砌）**
- 细线分割：1-2px 线条，主色或辅色，50% 透明度
- 色块背景：半透明色块（20% 透明度），用于强调区域
- 小图标：Material Design Icons 或 Feather Icons 风格
- 渐变背景：用于封面、章节分隔页

**输出**：
```python
{
  "page_number": 3,
  "layout": "text_left_image_right",
  "colors": {
    "primary": "#1A237E",    # 深蓝
    "secondary": "#00BCD4",  # 青绿
    "accent": "#FDD835",     # 高亮黄
    "text": "#424242",       # 深灰
    "bg": "#FFFFFF"          # 纯白
  },
  "fonts": {
    "title": {"size": 32, "bold": True, "color": "#1A237E"},
    "body": {"size": 20, "bold": False, "color": "#424242"},
    "note": {"size": 14, "bold": False, "color": "#757575"}
  },
  "decoration": {
    "has_divider": True,
    "divider_color": "#00BCD4",
    "divider_width": 2,
    "has_background_shape": False
  }
}
```

#### 阶段 5：配图生成
**目标**：调用 AI 图片生成为需要图片的页面生成专属配图。

**5.1 图片类型分类**
- **抽象概念图**：科技、创新、团队、成长等抽象概念（几何、渐变、线条）
- **商务场景图**：办公、会议、数据、团队协作（商务摄影风格）
- **数据可视化**：图表、趋势、对比（扁平化图表风格）
- **产品展示**：产品特写、功能展示（干净背景、聚焦产品）
- **背景纹理**：封面、章节页背景（低对比度、不抢眼）

**5.2 Prompt 生成策略**
- 根据页面内容和主题，生成精准的英文 prompt
- 风格后缀：`minimalist, clean background, professional photography style, high quality, 4K`
- 色调控制：`color palette: dark blue and gold` / `warm tones` / `cool tones`
- 构图控制：`centered composition` / `asymmetric layout` / `top-down view`

**5.3 图片尺寸**
- 横屏图片：1920x1080（适合左右分栏）
- 竖屏图片：1080x1920（适合封面、全屏页）
- 方形图片：1024x1024（适合图标、小配图）

**输出**：
- 为每个 `needs_image=True` 的页面生成图片，保存到 `workspace/ppt_xxx/images/` 目录
- 更新页面数据，加入图片路径：`"image_path": "workspace/ppt_xxx/images/slide_3.png"`

#### 阶段 6：PPT 渲染
**目标**：使用 python-pptx 生成最终 .pptx 文件，并进行质量检查。

**6.1 渲染逻辑**
- 创建空白演示文稿（16:9 宽屏）
- 设置全局主题（字体、配色）
- 逐页渲染：
  - 根据 layout 类型调用对应的渲染函数
  - 插入文本框（标题、正文、要点）
  - 插入图片（调整尺寸、位置、裁剪）
  - 添加装饰元素（线条、色块、图标）
  - 设置页面动画（可选，默认无动画，避免花哨）
  - 添加演讲备注（Speaker Notes）

**6.2 质量检查（自动）**
- **文本溢出检测**：检测文本框是否超出边界
- **元素重叠检测**：检测文本框、图片是否重叠
- **对齐检查**：检测元素是否按网格对齐
- **配色一致性**：检测全篇是否使用同一色板
- **图片缺失检测**：检测 `needs_image=True` 但图片未生成的页面

**6.3 修复策略**
- 文本溢出 → 缩小字号（最多 -4pt）或删减文字
- 元素重叠 → 调整位置（向下或向右偏移）
- 对齐问题 → 自动对齐到 8px 网格
- 配色不一致 → 警告日志，不自动修复（需要重新生成）
- 图片缺失 → 用纯色占位块 + 警告日志

**输出**：
- 最终 .pptx 文件：`workspace/ppt_xxx/output.pptx`
- 质量报告：`workspace/ppt_xxx/quality_report.json`

### 2.3 使用方式

#### 2.3.1 CLI 命令行
```bash
# 基本用法
python main.py ppt generate input/report.txt

# 指定主题和页数上限
python main.py ppt generate input/report.txt --theme modern --max-pages 15

# 使用品牌模板（自定义配色、字体、Logo）
python main.py ppt generate input/report.txt --brand-template company_brand.yaml

# 断点续传（生成失败后继续）
python main.py ppt generate input/report.txt --resume

# 查看进度
python main.py ppt status workspace/ppt_xxx

# 预览大纲（不生成图片和 PPT，仅输出大纲）
python main.py ppt outline input/report.txt
```

## 3. 非功能需求

### 3.1 性能要求
- **生成速度**：15 页 PPT，含配图，5-8 分钟内完成（取决于 LLM 和图片生成速度）
- **断点续传**：任何阶段失败后可恢复，不重复已完成的工作
- **并行优化**：配图生成可并行（最多 3 张同时生成，避免 API 限流）

### 3.2 兼容性
- **输出格式**：.pptx（Office Open XML 格式）
- **软件兼容**：PowerPoint 2016+、Keynote、WPS Office
- **字体 Fallback**：
  - Windows：微软雅黑 → Arial
  - macOS：苹方-简 (PingFang SC) → Helvetica
  - Linux：Noto Sans CJK SC → DejaVu Sans

### 3.3 可扩展性
- **插件系统**（未来）：支持自定义布局类型、自定义装饰元素
- **多语言**（未来）：英文 PPT 生成支持

### 3.4 错误处理
- **LLM 调用失败** → 重试 3 次，失败后保存 checkpoint，用户可选择换 LLM 继续
- **图片生成失败** → 用纯色占位块替代，标记为"待补图"，生成完成后可单独重新生成
- **文本溢出** → 自动缩小字号，仍溢出则截断并添加"..."
- **配色冲突** → 如果用户品牌色与主题冲突，优先使用品牌色

## 5. 验收标准

### 5.1 "精美"标准
- [ ] 配色方案不呆板（不是纯蓝白商务风）
- [ ] 字体层级清晰（至少 4 级）
- [ ] 留白充足（内容不超过页面 60%）
- [ ] 装饰元素微妙（不堆砌）
- [ ] 图片与内容高度相关（不是通用素材）

### 5.2 "有人味"标准
- [ ] 同一 PPT 中不连续 3 页使用相同布局
- [ ] 不是每页都是"标题+3 个 bullet point"
- [ ] 语言不刻板（避免"首先、其次、最后"）
- [ ] 数据具体化（不说"显著增长"，而是"增长 30%"）
- [ ] 有节奏感（文字页→图片页→数据页交替）

### 5.3 技术标准

## 6. 边界条件与约束

### 6.1 输入限制
- 文本长度：最少 500 字，最多 50000 字（超出则截断或分段）
- 文件格式：txt、markdown（未来支持 PDF、Word）
- 编码：UTF-8（其他编码自动检测并转换）

### 6.2 输出限制
- 页数：最少 5 页，最多 50 页（建议 8-30 页）
- 图片数量：每页最多 1 张主图 + 3 个小图标
- 文件大小：最终 .pptx 文件不超过 50MB（图片自动压缩）

### 6.3 依赖要求
- Python 3.10+
- python-pptx 库（PPT 生成）
- PIL/Pillow（图片处理）
- 现有 LLM 后端（OpenAI / DeepSeek / Gemini / Ollama）
- 现有图片生成后端（SiliconFlow / 阿里云万相 / Together.ai / diffusers）

## 7. 风险与缓解

### 7.1 风险 1：AI 配图相关性不高
**描述**：生成的图片与页面内容不匹配，显得"驴唇不对马嘴"。
**缓解措施**：
- Prompt 生成阶段，让 LLM 深度理解页面内容，生成精准的图片描述
- 添加"相关性检查"Agent（可选），用 Vision LLM 检测图片是否匹配内容，不匹配则重新生成
- 提供"无配图模式"选项，用户可选择纯文字 PPT

### 7.2 风险 2：生成速度慢
**描述**：15 页 PPT 需要 10-15 分钟，用户等待体验差。
**缓解措施**：
- 图片生成并行（最多 3 张同时生成）
- 提供"快速模式"（跳过配图生成，仅用纯色占位）
- Web UI 显示详细进度（"正在生成第 3 页配图..."）

### 7.3 风险 3：文本溢出
**描述**：要点文字太长，超出文本框边界。
**缓解措施**：
- 内容创作阶段，LLM 严格控制要点长度（每条不超过 30 字）
- 渲染阶段，自动检测并缩小字号（最多 -4pt）
- 仍溢出则截断并添加"..."，同时记录到质量报告中

### 7.4 风险 4：风格不统一
**描述**：有些页面像商务风，有些像创意风，整体不协调。
**缓解措施**：
- 设计编排阶段，全局使用同一套配色方案和字体
- 质量检查阶段，检测配色一致性，不一致则警告
- 提供"风格锁定"选项，用户可强制所有页面使用同一布局系列
