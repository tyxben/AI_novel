# AI PPT 生成 V2 - 需求文档（产品修正版）

## 1. 修正背景

### 1.1 当前问题诊断
当前 V1 版本存在以下产品定位偏差：
1. **假设错误的使用场景**：当前假设用户有完整文档需要"转化"为 PPT，但实际最高频场景是用户只有主题+零散想法，需要 AI 从零构思
2. **缺少人工介入点**：全自动黑盒流程，用户无法在关键环节（大纲确认）进行审核和调整
3. **资源投入失衡**：过度投入渲染层（48KB renderer、12 种布局），但 PPT 视觉不是 AI 核心优势；应强化内容质量
4. **输出形式单一**：仅输出 .pptx 文件，但用户可能需要 Markdown 大纲、演讲稿等中间产物

### 1.2 V2 核心修正策略
基于以上问题，V2 版本的四个核心调整：

| 调整方向 | 具体措施 | 优先级 |
|---------|---------|-------|
| **新增"从主题生成"入口** | 用户只需提供：主题、受众、场景、页数、风格、已有材料（可选），AI 从零构思内容结构 | P0 |
| **加入大纲审核环节** | 生成大纲后暂停，返回可编辑的大纲给用户，确认后再继续后续流程 | P0 |
| **简化渲染，强化内容** | 渲染保持"能用"即可（不再追加新布局），把力气花在内容质量（叙事弧线、演讲备注、配图建议）上 | P0 |
| **输出中间产物** | 除了 .pptx，还输出 Markdown 大纲、逐页内容文本、配图建议清单，用户可选择"只要内容不要 PPT" | P1 |

## 2. 功能需求

### 2.1 新增：从主题生成 PPT（主要入口）

#### 2.1.1 输入参数
用户仅需提供：
```python
{
    "topic": "2024 年度产品规划",  # 必填：主题（5-30 字）
    "audience": "business",  # 必填：受众类型（business/technical/educational/creative）
    "scenario": "quarterly_review",  # 必填：使用场景（详见场景枚举）
    "target_pages": 15,  # 可选：目标页数（默认 10-20）
    "theme": "modern",  # 可选：视觉风格（默认 modern）
    "materials": [  # 可选：用户提供的零散材料
        {"type": "text", "content": "Q1 GMV 增长 30%"},
        {"type": "text", "content": "用户反馈：结账流程太复杂"},
        {"type": "file", "path": "data/q1_report.txt"},
    ],
}
```

#### 2.1.2 场景枚举（Scenario）
| 场景 ID | 中文名称 | 典型页数 | 叙事结构 |
|--------|---------|---------|---------|
| `quarterly_review` | 季度汇报 | 12-18 | 背景→进展→数据→风险→下季计划 |
| `product_launch` | 产品发布 | 10-15 | 痛点→方案→功能→案例→CTA |
| `tech_share` | 技术分享 | 15-25 | 背景→原理→实践→案例→总结 |
| `course_lecture` | 课程讲义 | 20-30 | 目标→概念→举例→练习→回顾 |
| `pitch_deck` | 融资路演 | 8-12 | 问题→方案→市场→团队→融资 |
| `workshop` | 工作坊 | 15-20 | 破冰→学习→实践→分享→总结 |
| `status_update` | 项目进展 | 8-12 | 背景→进展→问题→计划→资源 |

#### 2.1.3 AI 构思流程
```mermaid
graph LR
    A[主题+受众+场景] --> B[叙事结构规划]
    B --> C[信息点挖掘]
    C --> D[整合用户材料]
    D --> E[生成大纲]
    E --> F[返回用户审核]
```

**步骤说明**：
1. **叙事结构规划**：根据场景选择叙事弧线（例如季度汇报用"背景→进展→数据→风险→计划"）
2. **信息点挖掘**：LLM 基于主题和场景，生成需要覆盖的关键信息点（例如"Q1 GMV 数据、用户增长、竞品动态"）
3. **整合用户材料**：将用户提供的零散材料（如果有）归类到对应信息点
4. **生成大纲**：每页分配 PageRole、标题草案、要点框架
5. **返回用户审核**：暂停，等待用户确认或编辑

#### 2.1.4 与"文档转 PPT"的对比
| 维度 | 从主题生成（新增） | 从文档转 PPT（V1） |
|------|------------------|-------------------|
| **输入** | 主题+受众+场景+零散材料 | 完整文档（txt/md） |
| **AI 职责** | 从零构思内容结构 | 提炼已有文档 |
| **适用场景** | 用户还没写稿，需要 AI 帮构思 | 用户有完整文档，需要转化为 PPT |
| **频率预估** | 70% | 30% |

**决策**：V2 将"从主题生成"作为**默认入口**，"文档转 PPT"降级为备选模式。

### 2.2 新增：大纲审核环节（人工介入点）

#### 2.2.1 流程变更
**V1 流程（黑盒）**：
```
文档输入 → 分析 → 大纲生成 → 内容创作 → 设计 → 配图 → 渲染 → 输出 .pptx
```

**V2 流程（暂停审核）**：
```
主题输入 → 叙事规划 → 大纲生成 → [暂停：用户审核] → 内容创作 → 设计 → 配图 → 渲染 → 输出
```

#### 2.2.2 大纲数据结构（可编辑）
```python
{
    "total_pages": 15,
    "estimated_duration": "8-10 分钟演讲",
    "narrative_arc": "quarterly_review",  # 叙事结构
    "slides": [
        {
            "page_number": 1,
            "role": "cover",  # PageRole 枚举
            "title": "2024 年度产品规划",  # 可编辑
            "subtitle": "技术驱动，用户至上",  # 可编辑
            "key_points": [],  # 封面页无要点
            "layout": "title_hero",  # 可编辑（从 LayoutType 枚举选择）
            "image_strategy": "illustration",  # 可编辑
            "speaker_notes_hint": "自我介绍 + 开场白",  # 可编辑
            "editable": True,  # 标记此页可编辑
        },
        {
            "page_number": 2,
            "role": "executive_summary",
            "title": "核心亮点",  # 可编辑
            "key_points": [  # 可编辑（增删改）
                "GMV 同比增长 30%",
                "用户满意度提升至 8.5 分",
                "推出 3 个重点新功能",
            ],
            "layout": "bullet_with_icons",
            "image_strategy": "none",
            "speaker_notes_hint": "快速总结本季度成绩，引出后续详细汇报",
            "editable": True,
        },
        # ... 更多页面
    ]
}
```

#### 2.2.3 用户可编辑操作
| 操作 | 说明 | 实现方式 |
|------|------|---------|
| **删除页面** | 删除不需要的页 | 从 slides 列表移除 |
| **新增页面** | 在某页后插入新页 | 指定插入位置、role、标题 |
| **调整顺序** | 调换页面顺序 | 重排 page_number |
| **修改标题** | 改标题文字 | 直接编辑 title 字段 |
| **修改要点** | 增删改要点 | 直接编辑 key_points 列表 |
| **更换布局** | 换成其他布局类型 | 从 LayoutType 枚举选择 |
| **修改配图策略** | 改为需要图表/示意图/不配图 | 从 ImageStrategy 枚举选择 |

#### 2.2.4 三端实现方式
| 端 | 实现方式 |
|----|---------|
| **CLI** | 生成大纲后，输出 JSON 文件到 `workspace/ppt_xxx/outline_editable.json`，用户手动编辑后运行 `python main.py ppt continue <project_path>` |
| **Web UI** | 显示可视化大纲编辑器（表格或卡片列表），用户在线编辑后点击"确认继续" |
| **MCP** | 新增 `ppt_review_outline(project_id)` 工具返回大纲，AI 助手可代替用户编辑，然后调用 `ppt_confirm_outline(project_id, edited_outline)` |

### 2.3 强化：内容质量提升（而非视觉渲染）

#### 2.3.1 资源重新分配
**V1 资源分配**：
- 渲染层：40%（12 种布局 × 复杂渲染逻辑）
- 内容层：30%（标题+要点提炼）
- 配图层：30%（AI 图片生成）

**V2 资源分配**：
- 渲染层：20%（保持现有 12 种布局，不追加新布局）
- 内容层：50%（强化内容质量，详见下文）
- 配图层：20%（简化为配图建议，可选生成）
- 输出层：10%（中间产物输出）

#### 2.3.2 内容质量强化点
| 强化项 | V1 现状 | V2 提升 | 实现方式 |
|-------|---------|---------|---------|
| **叙事弧线** | 无明确叙事结构，页面排列随意 | 根据场景选择叙事结构（起承转合、问题-方案、时间线等） | 新增 `NarrativeDesigner` Agent |
| **结论先行** | 习惯"背景→分析→结论"顺序 | 商务场景强制"结论先行"（Executive Summary 前置） | Outline 生成规则强化 |
| **演讲备注** | 仅生成简单备注 | 为每页生成详细演讲稿（200-300 字），包含开场白、过渡句、强调点 | `ContentCreator` 增加 speaker_notes 生成 |
| **配图建议** | 直接调用图片生成 | 先生成"配图建议"（类型、内容、构图），用户可选择是否生成 | 新增 `ImageAdvisor` 模块 |
| **要点精炼** | 要点可能冗长 | 每条要点不超过 20 字，避免句号结尾（显得像句子） | LLM prompt 强化 + 后处理检查 |
| **AI 味检测** | 无检测机制 | 检测"赋能"、"闭环"、"抓手"等空洞词，强制替换 | 黑名单词库（复用小说模块的 AI 味黑名单）|

#### 2.3.3 新增：配图建议（而非直接生成）
**V1 流程**：
```
确定需要配图 → 生成 prompt → 调用图片生成 API → 插入 PPT
```

**V2 流程**：
```
确定需要配图 → 生成配图建议 → 用户选择是否生成 → （可选）调用 API → 插入 PPT
```

**配图建议数据结构**：
```python
{
    "page_number": 3,
    "title": "市场规模持续增长",
    "image_suggestion": {
        "type": "chart",  # chart / diagram / illustration / ui_mock
        "description": "柱状图：2022-2024 市场规模对比",
        "composition": "3 根柱子，从左到右递增，突出 2024 年最高",
        "color_scheme": "蓝色系渐变",
        "alternative_text": "如果不生成图片，建议手动绘制柱状图或使用 Excel 导入",
        "prompt_if_generate": "A clean bar chart showing market size growth from 2022 to 2024, blue gradient bars, minimalist style, white background",
    },
    "generate_image": False,  # 默认不生成，用户可选择生成
}
```

**好处**：
1. 省钱：用户可以只要建议，不要图片（节省图片生成 API 费用）
2. 灵活：用户可以手动绘制更精准的图表（尤其是数据图表，AI 生成不如手动）
3. 透明：用户知道为什么要配这张图，而不是 AI 黑盒决策

### 2.4 新增：输出中间产物（不只是 .pptx）

#### 2.4.1 输出清单
| 输出文件 | 格式 | 说明 | 使用场景 |
|---------|------|------|---------|
| **final.pptx** | .pptx | 最终 PPT 文件 | 直接演示 |
| **outline.md** | Markdown | 大纲（带层级结构） | 用于 Canva / Gamma / 飞书文档 |
| **content.txt** | 纯文本 | 逐页内容（标题+要点+演讲稿） | 用于演讲排练 |
| **speaker_notes.pdf** | PDF | 演讲者备注（每页一页） | 打印后演讲时参考 |
| **image_suggestions.json** | JSON | 配图建议清单 | 用于手动制作配图 |
| **quality_report.md** | Markdown | 质量检查报告 | 了解哪些页面需要调整 |

#### 2.4.2 Markdown 大纲格式
```markdown
# 2024 年度产品规划
**演讲时长**：8-10 分钟 | **页数**：15 页 | **风格**：modern

---

## 第 1 页：封面
**标题**：2024 年度产品规划
**副标题**：技术驱动，用户至上
**布局**：title_hero
**配图建议**：抽象科技插图

**演讲稿**：
大家好，今天我将向大家汇报 2024 年度的产品规划...

---

## 第 2 页：核心亮点
**标题**：核心亮点
**要点**：
- GMV 同比增长 30%
- 用户满意度提升至 8.5 分
- 推出 3 个重点新功能

**布局**：bullet_with_icons
**配图建议**：无

**演讲稿**：
首先快速总结一下本季度的三大亮点...

---

## 第 3 页：市场规模持续增长
...
```

#### 2.4.3 用户选择"只要内容不要 PPT"
**CLI 选项**：
```bash
python main.py ppt generate input.txt --content-only
# 输出：outline.md + content.txt + image_suggestions.json
# 跳过：配图生成 + PPT 渲染
```

**Web UI**：
- 新增复选框："仅生成内容大纲（不渲染 PPT）"
- 勾选后，跳过渲染步骤，直接输出 Markdown

**适用场景**：
- 用户想用 Canva / Gamma 等在线工具自己排版
- 用户只需要演讲稿和大纲，不需要 PPT 文件
- 快速生成内容框架，后续在 PowerPoint 中手动调整

## 3. 非功能需求

### 3.1 性能要求（调整）
| 指标 | V1 | V2 | 说明 |
|------|----|----|------|
| **生成大纲** | 无单独指标 | 1-2 分钟 | 包含叙事规划 + 信息点挖掘 |
| **生成完整内容** | 5-8 分钟（15 页） | 3-5 分钟（15 页） | 不生成图片时更快 |
| **渲染 PPT** | 包含在总时间内 | 1 分钟 | 简化渲染逻辑后更快 |
| **配图生成（可选）** | 必须生成 | 用户选择 | 不强制生成 |

### 3.2 兼容性（保持）
- 输出 .pptx 兼容 PowerPoint 2016+ / Keynote / WPS
- Markdown 输出兼容 Canva / Gamma / 飞书文档

### 3.3 可扩展性（新增）
- **叙事结构可扩展**：用户可自定义新场景的叙事弧线（YAML 配置）
- **AI 味黑名单可扩展**：用户可添加自己的敏感词列表

## 4. 用户故事（修正版）

### 故事 1：产品经理构思季度汇报（从主题生成）
**角色**：产品经理
**场景**：需要准备 Q1 季度汇报 PPT，但还没写稿，只有零散数据和想法
**操作流程**：
1. 打开 Web UI，选择"从主题生成"
2. 输入主题："Q1 产品进展汇报"，受众：business，场景：quarterly_review
3. 上传零散材料：`q1_data.txt`（包含 GMV、用户增长等数据）
4. AI 生成大纲（12 页），显示在页面上
5. 用户微调：删除第 7 页（不需要），修改第 3 页标题
6. 点击"确认继续"，AI 生成详细内容
7. 选择"仅生成内容大纲（不渲染 PPT）"，下载 `outline.md` 和 `content.txt`
8. 在飞书文档中复制 Markdown，调整格式后分享给团队

**验收标准**：
- [ ] 大纲结构符合季度汇报习惯（背景→进展→数据→风险→计划）
- [ ] 用户材料被正确整合到对应页面
- [ ] Markdown 大纲可直接在飞书文档中使用
- [ ] 演讲稿详细，能直接照着讲

### 故事 2：技术专家修改 AI 生成的大纲（人工介入）
**角色**：技术专家
**场景**：准备技术分享 PPT，需要对 AI 生成的大纲进行专业调整
**操作流程**：
1. CLI 运行 `python main.py ppt create "微服务架构实践" --scenario tech_share`
2. AI 生成大纲，保存到 `workspace/ppt_xxx/outline_editable.json`
3. 用户打开 JSON 文件，手动编辑：
   - 删除"什么是微服务"页（受众已懂基础知识）
   - 新增"踩坑案例"页（在第 8 页后插入）
   - 将第 5 页布局改为 `comparison`（对比单体架构 vs 微服务）
4. 运行 `python main.py ppt continue workspace/ppt_xxx`
5. AI 继续生成内容、配图建议、渲染 PPT
6. 下载 `final.pptx` 和 `image_suggestions.json`
7. 根据配图建议，手动绘制架构图（AI 生成的架构图不够精准）

**验收标准**：
- [ ] JSON 编辑后，后续流程正确继承修改
- [ ] 新增页面的内容符合用户意图
- [ ] 配图建议实用（而非 AI 黑盒生成不相关的图）
- [ ] 用户可以选择部分页面生成图片，部分手动绘制

### 故事 3：创业者快速制作融资路演（只要内容）
**角色**：创业者
**场景**：需要准备投资人路演 PPT，时间紧迫，只需要内容框架
**操作流程**：
1. CLI 运行 `python main.py ppt create "AI 智能客服 SaaS" --scenario pitch_deck --content-only`
2. AI 生成大纲（10 页），包含：问题→方案→市场→团队→融资
3. 用户审核大纲，修改第 6 页标题（"我们的优势"→"核心壁垒"）
4. 确认后，AI 生成详细内容，输出 `outline.md` + `content.txt` + `image_suggestions.json`
5. 用户打开 Canva，导入 Markdown，使用 Canva 模板排版
6. 根据配图建议，在 Canva 中插入配图（或使用 AI 生成）
7. 1 小时内完成精美 BP，发送给投资人

**验收标准**：
- [ ] 大纲结构符合融资路演习惯（问题→方案→市场→团队→融资）
- [ ] 演讲稿有说服力，数据具体（不说"市场广阔"，而是"市场规模 100 亿"）
- [ ] Markdown 可直接导入 Canva
- [ ] 配图建议具体（"抽象科技背景"→"深蓝色渐变背景，右下角有金色曲线"）

### 故事 4：教师使用文档转 PPT（传统入口）
**角色**：大学教师
**场景**：已有课程讲义（10000 字 Markdown），需要转化为 PPT
**操作流程**：
1. Web UI 选择"从文档生成"，上传 `lecture.md`
2. AI 分析文档，生成大纲（25 页）
3. 用户审核大纲，调整部分页面顺序
4. 确认后，AI 生成内容 + 配图建议
5. 用户选择"生成所有配图"（因为是教学插图，AI 生成可接受）
6. 渲染 PPT，下载 `final.pptx` + `speaker_notes.pdf`
7. 打印演讲者备注，上课时参考

**验收标准**：
- [ ] 文档转 PPT 流程保持稳定（V1 功能不受影响）
- [ ] 大纲审核环节正常工作
- [ ] 演讲者备注 PDF 排版清晰，可打印

## 6. 边界条件与约束

### 6.1 输入限制
| 项目 | V1 | V2 | 说明 |
|------|----|----|------|
| **主题长度** | 无限制 | 5-30 字 | 太短无法构思，太长不是主题 |

## 7. 风险与缓解

### 7.1 风险 1：从主题生成质量不可控
**描述**：AI 从零构思内容，可能偏离用户期望。
**缓解措施**：
- 大纲审核环节让用户尽早介入
- 提供"零散材料"输入，引导 AI 生成方向
- 提供叙事结构预览，用户可切换场景（例如从 quarterly_review 换成 status_update）

### 7.2 风险 2：大纲编辑体验差
**描述**：JSON 编辑（CLI）或简陋表格（Web UI）体验不佳。
**缓解措施**：
- Web UI 设计专门的大纲编辑器（拖拽排序、卡片式编辑）
- CLI 提供 `outline.yaml` 替代 JSON（更易读）
- MCP 模式由 AI 助手代替用户编辑，无需人工操作 JSON

### 7.3 风险 3：配图建议不实用
**描述**：AI 生成的配图建议太抽象，用户无法手动制作或 AI 生成。
**缓解措施**：
- Prompt 强化：要求具体描述（"柱状图：3 根柱子，从左到右递增"）
- 提供 alternative_text（如果不生成图片，建议如何手动制作）
- 可选生成：用户可以逐页选择是否生成图片，不满意则手动替换

### 7.4 风险 4：演讲稿 AI 味重
**描述**：演讲稿充斥"赋能"、"闭环"等空洞词。
**缓解措施**：
- 复用小说模块的 AI 味黑名单（`src/novel/templates/ai_taste_blacklist.py`）
- 生成后自动检测并替换
- 提供"演讲稿风格"选项（正式/轻松/幽默）
