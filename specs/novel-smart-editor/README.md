# 小说智能编辑系统规格文档

## 概述
本目录包含小说智能编辑系统的完整规格文档，旨在将当前嵌入在 Web UI 中的编辑逻辑抽取为独立服务层，提供自然语言编辑能力，支持三端（Web UI / MCP / CLI）共用。

## 核心特性
- ✅ **自然语言编辑** — 用户说"添加角色李明"，系统自动生成完整角色档案
- ✅ **版本化设定** — 同一实体可在不同章节有不同状态（如角色成长）
- ✅ **影响分析** — 自动检测设定变更对已写章节的影响，提供修复建议
- ✅ **变更历史** — 记录所有修改，支持审计和回滚
- ✅ **三端共用** — Web UI、MCP Server、CLI 调用同一服务层

## 核心设计决策

### 决策 1：版本化设定而非替换
**理由**：允许同一角色在不同章节有不同状态（如第10章性格大变），保持历史连续性。

**实现**：
- 每个实体添加 `effective_from_chapter`, `deprecated_at_chapter`, `version` 字段
- 查询时根据章节号返回正确版本（`get_setting_at_chapter()`）

### 决策 2：服务层抽取而非重写 Web UI
**理由**：渐进式重构，降低风险，保证向后兼容。

**实现**：
- P0 阶段：Web UI 调用服务层，保留现有表单逻辑
- P1 阶段：重构表单编辑为调用服务层
- P2 阶段：完全移除 Web UI 中的 JSON 操作

### 决策 3：自然语言 + 结构化双轨并行
**理由**：自然语言降低学习成本，结构化编辑保证精确性。

**实现**：
- `IntentParser` 支持自然语言解析
- `NovelEditService.edit()` 同时接受 `instruction` 和 `structured_change`
- Web UI 表单编辑转换为结构化指令

### 决策 4：乐观锁而非悲观锁
**理由**：Web UI 多标签页场景下，悲观锁会导致用户体验差。

**实现**：
- 基于 `updated_at` 时间戳检查
- 冲突时友好提示用户刷新
- P2 可选增强：文件锁（适用于 CLI 场景）

## 关键风险与缓解

### 风险 1：LLM 解析不稳定
**影响**: 用户指令复杂时可能误解，导致错误修改
**缓解**:
- 重试机制（最多 3 次）
- 规则解析器兜底
- Dry run 模式预览（用户确认）

### 风险 2：向后兼容性破坏
**影响**: 老项目加载失败
**缓解**:
- 所有新字段有默认值
- 迁移脚本（自动升级）
- P0 测试老项目加载

### 风险 3：并发编辑冲突
**影响**: 多用户同时编辑导致数据损坏
**缓解**:
- 乐观锁（基于 updated_at）
- 友好错误提示
- P2 可选文件锁
